# Contributing

Thanks for your interest in contributing!

## Workflow

1. Fork the repository.
2. Create a feature or fix branch off `main`.
3. Make your changes.
4. Test with Waybar on a live Hyprland session before submitting.
5. Open a pull request against `main`.

## Guidelines

- **Keep it simple (KISS).** Small, focused PRs are easier to review and merge.
- **Shell scripts must remain bash-compatible.** Use `#!/usr/bin/env bash` and avoid bashisms that only work in zsh or fish.
- **Test before you submit.** Make sure the toggle and restart scripts work correctly with your Waybar config.

## Reporting Issues

If you find a bug or have a feature request, open an issue using the provided templates.
